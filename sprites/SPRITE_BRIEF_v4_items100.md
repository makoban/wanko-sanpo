# 発注書 v4 — アイテム100種＋おせわ用キャラ（gpt-image2）—「わんこさんぽ」

## 目的
ゲームを「たまごっち風の犬のおせわ」へ進化させる。
- 釣った魚・虫・くだもの・たからものを**それぞれ固有の絵**で見せる（図鑑・ポケット・売買画面用）
- **エサやり・うんち片付け・シャンプー・看病**などのおせわ素材
- 犬の成長（こいぬ→おとな→シニア）と、ぐあいがわるい時のポーズ

生成は **1枚 = 1アイテム**（シートにしない）。全100種＋犬の追加シート。

---

## 🎨 共通ルール（全アイテム厳守）

1. **1画像に1アイテムだけ**。中央に大きく（画面の70%くらい）
2. v3キャラと同じドット絵スタイル：
   - 正確なピクセル格子、アンチエイリアス・グラデ禁止、ベタ塗り
   - こげ茶 #4a3228 の1pxアウトライン
   - パステル基準色：クリーム#fff3e0 / ピンク#ff8fae / 緑#8ecf6f / 水色#aee3f5 / 黄#ffd166
   - **32×32ドット相当**の粗さで（細かくしすぎない）
3. **いきもの（さかな・むし・カメ等）はかわいい目をつける**：黒目＋白キャッチライト。魚は横向き
4. **もの（道具・食べ物）は顔をつけない**。ただし丸く・ぽってり・かわいいフォルムで
5. 背景は**無地の白**（透過処理はこちらでやるので市松模様は不要）
6. 各カテゴリの**1個目を最初に作り、以降はそれを参照画像に添付**して統一
   （全体のタッチは `sprites/mimii.png`＝v3基準も毎回添付）
7. 影・地面は描かない。アイテム単体で浮いている状態

## 📁 保存場所（厳守）

```
sprites/items_raw/<カテゴリ>/<ファイル名>.png   ← gpt-image2の出力をそのまま保存
```
カテゴリ = fish / bug / fruit / food / care / treasure / toy / fashion / icon
※ 32×32への変換と透過化はClaude側で行い `sprites/items/<カテゴリ>/` に置くので、
   Codexは items_raw に置くだけでよい。ファイル名は下の表の通り**正確に**。

## 🛠 制作手順

1. カテゴリごとに順番に作る（fish → bug → fruit → food → care → treasure → toy → fashion → icon → dogchar）
2. **各カテゴリの1個目**（funa / monshiro / ringo / dogfood / poop / kaseki / tennis-ball / bandana-aka / heart）を最初に作り、
   以降はそのカテゴリ1個目＋mimii.pngを参照画像に添付して統一
3. 1個できるごとにチェック → NGは再生成：
   - [ ] 1アイテムだけ・中央・大きい
   - [ ] ドット絵（AAなし）・こげ茶アウトライン・パステル
   - [ ] いきものはキャッチライト付きの目、ものは顔なし
   - [ ] 背景は無地
   - [ ] ファイル名が表と一致
4. 全部できたら、カテゴリごとの完成ファイル一覧を報告

## 🎮 このアセットで作る予定のあそび（参考・実装はClaude担当）

- **おなかメーター**：時間でへる → ごはん（bowl-full）をあげる。りんごや魚もおやつにできる
- **うんち**：ときどき地面に出る → スコップで片付け。放置すると…
- **ごきげん・けんこう**：空腹放置・うんち放置で低下 → sick状態（pose_sick＋sickアイコン）→ おくすりで看病
- **看病しないと**…天使のはね（hane.png）でおわかれ。おはかではなく「お空からまた会いに来る」やさしい演出＋あたらしい こいぬ（dog_puppy）を迎える
- **成長**：こいぬ→おとな→シニア。ごはんとお世話の積み重ねで成長
- 図鑑・ポケット・商店の絵文字を全部このアイコンに差し替え
